# Risk Model

QA Cortex evaluates business impact before reviewing technical quality.

---

# Risk Levels

## Critical

Examples:

- Login
- Authentication
- Payment
- Checkout
- Money Transfer

---

## High

Examples:

- Registration
- User Profile
- Password Reset
- File Upload

---

## Medium

Examples:

- Search
- Filters
- CRUD Operations
- Notifications

---

## Low

Examples:

- Theme Selection
- UI Preferences
- Static Pages

---

# Risk Principles

Higher risk requires:

- More negative tests
- More boundary tests
- More security validation
- More assertions
- Better coverage
